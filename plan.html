Trading Indicator Notification System

System Blueprint: Approved Concrete Production Stack

Language: Go Data Provider: Alpaca API Local Cache: SQLite (WAL) Alerts: Discord Webhooks

1. Verified Problem Space & Constraints

  • Core Engine: High-frequency (hourly) evaluation of 50+ stocks across a historical depth of up to 1-2 years.
  • Black-Box Indicators: Formulas are nondisclosure. The infrastructure must provide a clean data array input wrapper to allow hot-swapping secret indicator algorithms.
  • Data Integrity: Uses Alpaca's adjusted historical close records to bypass corporate accounting distortions (such as stock splits and cash dividends).
  • API Rate Mitigation: Avoids simultaneous API hammering by deploying a scheduled worker pool to distribute and sequence requests over time.

2. Selected Tech Stack Implementations

Go (Concurrency & Scheduling Engine)

Uses native channels to build a controlled worker pool. Instead of blasting 50 requests simultaneously, a fixed number of worker goroutines handle a shared job queue sequentially. This keeps execution stable and safely under the free-tier rate limits (200 requests/minute).

Alpaca Market Data API (Ingestion Feed)

Provides robust 1-hour interval bars. Serves pre-adjusted OHLCV datasets natively, eliminating the need to write backend data-smoothing layers for splits or dividends.

SQLite (State Storage & Rolling Cache)

Acts as the local persistent cache. Database connection must use PRAGMA journal_mode=WAL; (Write-Ahead Logging). This allows background data-ingestion workers to write new hourly data safely without blocking the concurrent reading operations running the indicators.

3. The Configurable Lookback Cascade

To support varying indicators without forcing excessive disk I/O, data array sizes are determined by a cascading hierarchy:

Priority Configuration Level Scope & Function
1 (Highest) Stock Override Specific parameters assigned to volatile assets.
2 Indicator Requirement The native history required by the algorithm (e.g., 14 hours).
3 (Lowest) Global System Default Fallback window depth (defaulting up to 2 years).
